Privacy Policy

Last updated: June 16, 2026

The short version: this tool collects nothing. Every password is generated entirely inside your own browser and is never sent anywhere.

What we collect

Nothing. There are no accounts, no sign-ups, no analytics, no tracking pixels, no cookies, and no collection of any personal information.

How passwords are generated

Passwords are generated entirely in your browser (client-side) using the browser's built-in Web Crypto API for secure randomness. There is no server that receives, processes, or stores a generated password — in fact there is no application server at all. Nothing you generate is transmitted, logged, or seen by us or anyone else.

The only data stored anywhere

The optional Save history feature — which is off by default — stores your recent passwords in your own browser's local storage, on your device. This data:

  • never leaves your browser,
  • is never sent to us or to any third party, and
  • can be permanently deleted at any time using the Clear button in the History section.

If you never enable Save history, nothing is stored at all.

Cookies and third parties

This site sets no cookies and loads no third-party scripts, fonts, or services — everything, including the font, is served from this domain. This is enforced technically by a Content-Security-Policy that blocks the page from making any outbound network connection.

Changes to this policy

If this site's data practices ever change — for example, if advertising is introduced in the future — this policy will be updated and the "Last updated" date above will change.

Contact

Questions about privacy? Open an issue on the project's GitHub repository.
